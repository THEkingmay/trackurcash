Multi-Wallet: ระบบบันทึกรายรับ-รายจ่ายแบบหลายโปรไฟล์
1. บทนำ (Introduction)

Multi-Wallet เป็นเว็บแอปพลิเคชันสำหรับจัดการรายรับและรายจ่ายส่วนบุคคล โดยรองรับการใช้งานหลายโปรไฟล์ภายในบัญชีเดียว ผู้ใช้สามารถแยกข้อมูลทางการเงินของแต่ละโปรไฟล์ออกจากกันได้อย่างชัดเจน เหมาะสำหรับการใช้งานที่ต้องการแบ่งบัญชี เช่น การใช้ส่วนตัว ธุรกิจ หรือการใช้งานร่วมกัน

ระบบถูกออกแบบโดยคำนึงถึงความถูกต้องของข้อมูล (Data Integrity) ความรวดเร็วในการประมวลผล และความปลอดภัยในการเข้าถึงข้อมูลเป็นหลัก

2. วัตถุประสงค์ (Objective)
เพื่อพัฒนาเว็บแอปพลิเคชันสำหรับบันทึกและจัดการรายรับ-รายจ่าย
เพื่อรองรับการใช้งานหลายโปรไฟล์ภายใต้บัญชีเดียว
เพื่อให้ผู้ใช้สามารถวิเคราะห์พฤติกรรมทางการเงินของตนเองได้
เพื่อออกแบบระบบที่มีความปลอดภัยและมีประสิทธิภาพสูง
3. ขอบเขตของระบบ (Scope of Work)
3.1 ระบบจัดการตัวตน (Identity & Access Management)
รองรับการเข้าสู่ระบบด้วย Email และ Password
รองรับ Social Login ผ่าน Google โดยใช้ NextAuth.js
จัดการ Session อย่างปลอดภัย
ใช้ Upstash Redis สำหรับ Rate Limiting เพื่อป้องกันการโจมตีแบบ Brute Force
รองรับระบบ OTP สำหรับการยืนยันตัวตนและการรีเซ็ตรหัสผ่าน
ตรวจสอบความถูกต้องของข้อมูลด้วย Zod ทั้งฝั่ง Client และ Server
3.2 ระบบจัดการโปรไฟล์ (Multi-Profile Management)
ผู้ใช้หนึ่งบัญชีสามารถสร้างโปรไฟล์ย่อยได้หลายโปรไฟล์
ข้อมูลของแต่ละโปรไฟล์ถูกแยกออกจากกันอย่างชัดเจนด้วย profile_id
สามารถสลับโปรไฟล์ได้โดยไม่ต้องออกจากระบบ
3.3 ระบบบันทึกธุรกรรม (Transaction Management)
เพิ่ม แก้ไข และลบรายการรายรับและรายจ่าย
ข้อมูลที่จัดเก็บประกอบด้วย:
จำนวนเงิน
หมวดหมู่
รายละเอียดเพิ่มเติม
วันที่ทำรายการ
แสดงผลยอดคงเหลือรายวันแบบ Real-time
3.4 ระบบรายงานและวิเคราะห์ (Reporting & Analytics)
แสดง Dashboard สรุปข้อมูลตามช่วงเวลา:
รายวัน
รายเดือน
รายปี
แสดงการวิเคราะห์การใช้จ่ายตามหมวดหมู่ (Category Breakdown)
4. รายละเอียดทางเทคนิค (Technical Specifications)
ส่วนประกอบ	เทคโนโลยี	เหตุผลที่เลือกใช้
Framework	Next.js (App Router)	รองรับ Server Actions และ SSR ที่มีประสิทธิภาพ
ภาษา	TypeScript	ลดข้อผิดพลาดใน Runtime และเพิ่มความปลอดภัยของชนิดข้อมูล
ฐานข้อมูล	Supabase (PostgreSQL)	รองรับการขยายตัวและมี Real-time Feature
ORM	Drizzle ORM	Lightweight และ Type-safe
การออกแบบ UI	Tailwind CSS	พัฒนา UI ได้รวดเร็วและเป็นระบบ
Validation	Zod	ตรวจสอบ Schema ได้ทั้งฝั่ง Client และ Server
Cache และ Security	Upstash Redis	เหมาะสำหรับ Rate Limiting และ OTP
5. จุดเด่นของระบบ (Key Highlights)
รองรับการใช้งานหลายโปรไฟล์ในบัญชีเดียว
มีการแยกข้อมูลอย่างชัดเจน ลดความเสี่ยงของข้อมูลปะปนกัน
มีระบบความปลอดภัย เช่น Rate Limiting และ OTP
แสดงผลข้อมูลแบบ Real-time
ใช้ Type-safe stack เพื่อลดข้อผิดพลาดในการพัฒนา
6. การนำไปใช้งาน (Use Cases)
ใช้บันทึกรายรับ-รายจ่ายส่วนบุคคล
ใช้แยกบัญชีค่าใช้จ่ายระหว่างงานและชีวิตส่วนตัว
ใช้ติดตามพฤติกรรมทางการเงินเพื่อวางแผนการใช้เงิน
